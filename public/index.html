Tailstart

Components ready to style with Tailwind CSS

Usage

This index.html contains links to child HTML files, each containing HTML components.

You should edit the PostCSS files located in /src/css/ to edit the style, which by default are empty or with a few guidelines.

You can run the server using yarn dev or npm dev, and start editing the files. Your CSS changes will be instantly updated in your browser.

Alternatively, you can edit each HTML itself for using with Component Extraction later.

Happy coding!

Components

These are elaborated components which may be useful for your application, but you will need one or multiple classes.

  • Progress bar

    A bar denoting progress. Sometimes is controlled by Javascript.

  • Badge

    Visual indicators for labeling objects, or use for tagging.

  • Alert

    Blocks denoting warnings, danger, success, or other temporal information.

  • Card

    Self-contained information with their own context and actions.

  • List group

    Elaborated lists of information, each piece related to their neighbors.

  • Header

    Persistent header bar with links to navigate across a site.

  • Input group

    Inputs paired as groups with buttons, labels, checkboxes, inputs or radios.

  • Table

    Table head, body, footer, cells...

  • Navigation

    Lists with links to navigate through a section.

  • Breadcrumb

    Navigation bar that informs page hierarchy.

  • Pagination

    List of links to navigate along a paginated set of items.

  • Custom component

    You can add your own component here.
